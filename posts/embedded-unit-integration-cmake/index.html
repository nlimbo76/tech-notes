임베디드 유닛/통합 테스트 구축 실습 (Unity+CMock+CMake+QEMU)

개요#

가볍고 이식성 높은 Unity + CMock 조합과 CMake/CTest로 “호스트에서 빠른 유닛 테스트 → 선택적 QEMU 통합 스모크”를 만드는 실습 가이드입니다. 크로스 환경 의존을 최소화하고, HIL 전에 최대한 논리 버그를 제거하는 게 목표입니다.

1) 폴더 구조(예시)#

project/
 ├─ src/                # 제품 코드
 ├─ include/            # 헤더
 ├─ tests/
 │   ├─ unit/           # 유닛 테스트
 │   └─ integration/    # 통합 테스트 (QEMU 스모크 포함)
 ├─ third_party/
 │   ├─ unity/
 │   └─ cmock/
 └─ CMakeLists.txt

2) 의존성 (간단 설치)#

# Unity & CMock 가져오기 (서브모듈 권장)
git submodule add https://github.com/ThrowTheSwitch/Unity third_party/unity
git submodule add https://github.com/ThrowTheSwitch/CMock third_party/cmock

3) 유닛 테스트 샘플#

src/ringbuf.c를 예로:

// include/ringbuf.h
#pragma once
#include <stdint.h>
typedef struct { uint8_t* buf; int cap, head, tail; } ringbuf_t;
void rb_init(ringbuf_t* rb, uint8_t* mem, int cap);
int  rb_push(ringbuf_t* rb, uint8_t v);
int  rb_pop(ringbuf_t* rb, uint8_t* out);
int  rb_size(const ringbuf_t* rb);

// tests/unit/test_ringbuf.c
#include "unity.h"
#include "ringbuf.h"

static uint8_t mem[8];
static ringbuf_t rb;

void setUp(void){ rb_init(&rb, mem, 8); }
void tearDown(void){}

void test_push_and_pop(void){
  TEST_ASSERT_EQUAL_INT(0, rb_size(&rb));
  TEST_ASSERT_TRUE(rb_push(&rb, 0x11));
  TEST_ASSERT_EQUAL_INT(1, rb_size(&rb));
  uint8_t v=0;
  TEST_ASSERT_TRUE(rb_pop(&rb, &v));
  TEST_ASSERT_EQUAL_HEX8(0x11, v);
}

CMakeLists.txt(요지):

add_executable(test_ringbuf
  tests/unit/test_ringbuf.c
  src/ringbuf.c
  third_party/unity/src/unity.c
)
target_include_directories(test_ringbuf PRIVATE include third_party/unity/src)
add_test(NAME test_ringbuf COMMAND test_ringbuf)

4) CMock로 HAL 모킹#

하드웨어 접근 함수가 있는 hal_uart.h를 모킹:

// include/hal_uart.h
#pragma once
int hal_uart_write(const char* data, int len);

CMock로 모의 생성(빌드 스텝 or 사전 생성):

ruby third_party/cmock/vendor/cmock/lib/cmock.rb -o config.yml include/hal_uart.h
# → mocks/Mockhal_uart.c, Mockhal_uart.h 생성

테스트에서 사용:

#include "unity.h"
#include "Mockhal_uart.h"   // CMock 생성 헤더
#include "proto.h"          // 테스트 대상

void test_proto_send_hello(void){
  hal_uart_write_ExpectAndReturn("HELLO", 5, 5);
  TEST_ASSERT_TRUE(proto_send_hello());
}

5) QEMU로 통합 스모크(옵션)#

  • ARM Cortex-M 계열은 qemu-system-gnuarmeclipse 또는 qemu-system-arm 사용.
  • 부팅 메시지 / 특정 레지스터 초기화 여부 검증처럼 HIL 전 가벼운 연기.

예시(아이디어):

qemu-system-arm -M stm32-p103 -kernel build/firmware.elf -nographic -serial stdio | tee qemu.log
grep "Boot OK" qemu.log

6) CI 팁#

  • 유닛 테스트는 호스트 런타임으로 빠르게(초·분 단위).
  • CMock 생성은 캐시하거나 CI에서 Ruby 세팅을 넣어 반복 실행.
  • QEMU 스모크는 PR 라벨/경로 조건으로 선택 실행.
